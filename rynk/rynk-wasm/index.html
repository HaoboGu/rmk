Rynk web demo

The page (the example shell) owns the transport: it opens a link (USB Web Serial, or BLE via WebHID over the OS HID link), runs the permanent get_version probe, then dynamically loads the version-matched rynk-wasm package and drives the typed session over a JS bridge. Topic pushes (layer changes, WPM, …) stream live once connected — marked 📡 below. WebHID reaches the firmware's RynkHidService (vendor usage page 0xFF60) and rides the existing OS bond, so there is no pairing prompt.
