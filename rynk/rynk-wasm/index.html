Rynk web demo

The page (the example shell) owns the serial port: it opens it, runs the permanent get_version probe, then dynamically loads the version-matched rynk-wasm package and drives the typed session over a JS bridge. Topic pushes (layer changes, WPM, …) stream live once connected — marked 📡 below. (BLE configuration is native-only — use a native build, e.g. cargo run -p rynk --example hw_test -- ble.)
